# 手撕

## 大小端

``` c
bool isLittleEndian(){
    union{
        int num;
        char bytes[sizeof(int)];
    }test;
    
    test.num = 1;
    return test.bytes[0] == 1;
}

int main(){
    if(isLittleEndian())
        cout << "LittleEndian" << endl;
    else
        cout << "BigEndian" << endl;
    return 0;
}
```



# FreeRTOS

## 队列

- **消息以拷贝的形式入队，而不是以引用的形式**。**（安全大于性能）**
  - 如果传递引用，data 在函数栈上，函数返回后内存失效！
  - 拷贝机制让发送者可以立即释放资源（消息变量），而不必担心接收者使用时的内存安全问题。
  - 由于是拷贝，发送后对原数据的修改不会影响队列中已存储的消息。

- 为什么传入的是**指针常量**？

  - `void*` 是一个通用指针，可以指向任何类型的数据。
  - 使用`const void*`承诺不会通过这个指针修改源数据（只读）。
  - **在函数内部使用`memcopy`来完成拷贝。**

- 为什么在FreeRTOS中xQueueSendToFront()不能在中断中使用：

``` c
// 普通版本的可能行为
BaseType_t xQueueSendToFront(QueueHandle_t xQueue, 
                            const void *pvItemToQueue,
                            TickType_t xTicksToWait) {
    
    // 如果队列满了...
    if(队列满) {
        if(xTicksToWait > 0) {
            // 普通版本可能将任务置于阻塞状态！
            vTaskPlaceOnEventList(&xQueue->xTasksWaitingToReceive, xTicksToWait);
            taskYIELD(); // 引发任务调度！
        }
    }
    
    // 在中断中任务调度是绝对不允许的！
}
```

而在中断版本中：不阻塞，直接返回队列已满错误代码 `errQUEUE_FULL` 。
