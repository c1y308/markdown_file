# 汇编语言

汇编指令的最典型书写模式如下所示：

``` assembly
标号 操作码 操作数 1, 操作数 2, … ;注释
```

操作码是指令的助记符，它的前面必须有至少一个空白符，通常使用一至二个“Tab”键来产生。操作码后面往往跟随若干个操作数，而**第 1 个操作数，通常都给出本指令的执行结果存储处**。

立即数必须以“#”开头；还可以使用 EQU 指示字来定义常数，然后在代码中使用它们；注释均以”;”开头。

``` assembly
NVIC_IRQ_SETEN0 EQU 0xE000E100 ; 注意：常数定义必须顶格写
MOV R1, #NVIC_IRQ0_ENABLE ; 把立即数传送到 R1 中
```

# M核

## 寄存器

### 核心通用寄存器

所有 M 核均基于 ARMv6-M/ARMv7-M/ARMv8-M 架构，核心通用寄存器完全一致：

- 通用寄存器：R0~R12（32 位通用运算 / 数据寄存器）
- 专用寄存器：**R13 (SP，栈指针)、R14 (LR，链接寄存器)、R15 (PC，程序计数器)**
- 基础状态寄存器：PSR程序状态寄存器，(可细分为APSR, EPSR, IPSR)；存储运算标志位：N/Z/C/V。
- 基础中断寄存器：PRIMASK（全局中断屏蔽寄存器）

---

### 扩展寄存器

| 内核 / 架构                | 关键寄存器差异点                                             |
| :------------------------- | :----------------------------------------------------------- |
| **M0 / M0+ (ARMv6-M)**     | 基础配置：**无** `FAULTMASK`、`BASEPRI`（中断优先级屏蔽）寄存器；**无** MPU（内存保护单元）寄存器，**无** DSP指令。 |
| **M3 (ARMv7-M)**           | **新增** `FAULTMASK`、`BASEPRI`（精细化中断/故障屏蔽），支持完整 **MPU寄存器组**（通常管理8个内存区域），**新增**用于非对齐访问支持的 `CCR`（配置与控制寄存器）。 |
| **M4 (ARMv7E-M)**          | 在M3基础上，**新增单精度浮点单元寄存器** `S0-S31` 与 `FPSCR`；**新增** DSP扩展寄存器及状态位（如 `EPSR`中的 `Q`位）。 |
| **M7 (ARMv7E-M)**          | 寄存器集与M4类似。主要增强在**总线架构、缓存（含缓存维护操作相关的系统寄存器）及可选的双精度浮点**。 |
| **M23 (ARMv8-M Baseline)** | 相当于ARMv8-M版的M0+，**新增** TrustZone安全扩展寄存器（**SAU, IDAU**相关配置），提供比M0+更强的软件隔离基础。 |
| **M33 (ARMv8-M Mainline)** | 相当于ARMv8-M版的M4，**集成** TrustZone安全扩展（**SAU, IDAU**），安全状态区分**安全/非安全寄存器副本**（如 `CONTROL_S`, `CONTROL_NS`）。 |

1. **横向演进（功能增强）**
   - **从M0到M3**：是**从“无管理”到“有管理”** 的飞跃，增加了中断优先级管理和内存保护（MPU）的关键寄存器。
   - **从M3到M4/M7**：是**从“控制”到“信号处理”** 的扩展，增加了为浮点运算和数字信号处理（DSP）服务的专用寄存器。
2. **纵向升级（架构迭代）**
   - **从ARMv7-M到ARMv8-M**：最大的变革是引入了**TrustZone安全扩展**。M23/M33内核通过新增的安全配置寄存器（如SAU），在硬件上支持将代码、数据和外设划分为安全和非安全世界，这是实现现代物联网设备安全性的基石。

## AAPCS

**AAPCS**规定**单个形参直接通过 R0 寄存器传入**。如果任务函数有多个形参，则前 4 个形参会依次用 R0/R1/R2/R3 传递，超过 4 个的部分才会通过栈传递。

**寄存器分类（调用者 / 被调用者保存）**：

- 「调用者保存」：R0~R3（临时寄存器）—— 调用者如果需要保留这些值，必须自己先压栈，因此被调用函数可以随意修改；**R0~R3 是传参 / 临时运算的高频寄存器，被调用函数几乎一定会用**，调用者按需保存即可（比如只保存自己要用到的 R0）；（FreeRTOS任务切换时CPU**自动保存**这些寄存器的值到栈）
- 「被调用者保存」：R4~R11（非临时寄存器）—— 被调用函数如果要修改这些寄存器，必须先压栈，返回前恢复（FreeRTOS 任务切换时需要**手动保存**这些寄存器到栈）

---

**为什么不全是调用者保存寄存器？**

如果把 R0~R11 全部定为 “调用者保存”，意味着**每次调用函数时，调用者必须把所有寄存器的值都压栈保存**（不管被调用函数会不会用到这些寄存器），否则数据就可能被覆盖。

``` assembly
; 主函数（调用者）
main:
    PUSH {R4-R5, LR}        ; 保存被调用者保存寄存器(R4,R5)和返回地址(LR)
    MOV R4, #10             ; R4 = 10 (将被调用者保存寄存器用作局部变量)
    MOV R5, #20             ; R5 = 20
    MOV R0, R4              ; 第一个参数: R0 = 10
    MOV R1, R5              ; 第二个参数: R1 = 20
    
    ; 调用函数前，保存调用者保存寄存器
    MOV R2, #30             ; R2 = 30 (调用者保存寄存器，用于临时计算)
    MOV R3, #40             ; R3 = 40
    
    BL add_numbers          ; 调用函数
    ; 函数返回后，继续执行...
    
    POP {R4-R5, PC}         ; 恢复寄存器并返回

; 被调用函数
add_numbers:
    PUSH {R4-R5, R7}        ; 保存被调用者保存寄存器
    ADD R7, SP, #0          ; 设置帧指针（可选）
    
    ; 函数体
    ADD R0, R0, R1          ; R0 = R0 + R1
    ADD R0, R0, #100        ; R0 += 100
    
    POP {R4-R5, R7}         ; 恢复被调用者保存寄存器
    BX LR                   ; 返回调用者
```

在FreeRTOS中，每个任务的独立栈，就是该任务内部所有函数调用所共享的“舞台”，而每个函数被调用时，都会在这个舞台上临时搭建一个属于自己的“后台区域”——这就是**栈帧**。

`funcA` 调用 `funcB` 前，必须把 `R0~R11` 全部压栈（12 个寄存器 × 4 字节 = 48 字节），但 `funcB` 只改了 R0，R4~R11 全程没碰 —— 这 40 字节（R4~R11）的栈空间完全是 “白浪费”。

嵌入式系统（比如 Cortex-M）的栈空间通常只有几 KB（甚至几百字节），这种无意义的占用会大幅增加栈溢出风险。

funcA 要执行 12 次 `push` 指令（保存 R0~R11），调用完 funcB 后还要执行 12 次 `pop` 指令（恢复）；但实际只需要保存 R0 就够了，多出来的 22 条指令（11 个寄存器的 push+pop）都是 “无效操作”，会拖慢程序执行速度（尤其是中断、FreeRTOS 任务切换这种高频调用场景）。

---

**返回值规则**：

- 简单类型（int、char、指针、结构体指针）：通过 R0 返回；
- 复杂类型（大结构体）：调用者先在栈上分配空间，R0 指向该空间，被调用函数把返回值写入该空间。

## Cortex-M3

### Thumb-2 
Cortex-M3 处理器使用的是 **ARMv7-M 架构的 Thumb-2 指令集**。

- Thumb-2 是 ARM 的一种混合指令集，**融合了 16 位和 32 位指令**。
- 它继承了早期 Thumb 指令集的**高代码密度**（节省存储空间），同时通过加入 32 位指令实现了接近 ARM 32 位指令集的**高性能**。
- Cortex-M3 **只支持 Thumb-2**，无法执行传统的 ARM 32 位指令（如 ARMv7-A 的 ARM 模式指令）。
- 常用操作用 16 位指令，复杂操作用 32 位指令；早期 ARM 处理器需在 ARM/Thumb 模式间切换，Thumb-2 无需切换即可实现全部功能。

**Cortex-M0/M0+**：使用 ARMv6-M 架构的 **Thumb-1 指令集（部分 Thumb-2）**，仅支持 16 位指令和少量 32 位指令。

**Cortex-M4**：在 M3 基础上增加 **DSP 指令和可选单精度浮点单元**（FPU），适用于数字信号处理。

**Cortex-M7**：支持更复杂的双发射流水线和可选双精度浮点单元，性能更强。
